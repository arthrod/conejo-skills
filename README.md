# arthrod's Skill Registry

Curated personal skill set for Claude Code. The canonical source lives in
`~/.claude/skills/` on arthrod's machine; this repo is the published
mirror, regenerated by `scripts/update-skills.sh`.

**v2.0.0** — major restructure from ~163 unfocused skills down to ~94 curated
ones. Families collapsed (Pi → 2 skills, Pydantic AI → 1 with `references/`,
Rust → `rust-author` + `rust-review`, Better Auth slimmed, design dials folded
into `impeccable`), new combined skills (`conejo` absorbs code-review + autofix
+ pr-triage-gh, `marimo` absorbs 4 marimo skills, `ml-gpu-training` absorbs
CUDA setup). All design tools include a Stitch-first mandate (`STITCH-DESIGN.md`
in each design-skill folder).

## Sync workflow

```bash
# Regenerate this folder from ~/.claude/skills/ on a random new branch, commit, push:
./scripts/update-skills.sh

# Same but local-only (no commit, no push) — review before shipping:
DRY_RUN=1 ./scripts/update-skills.sh

# Different source directory:
SOURCE=/path/to/skills ./scripts/update-skills.sh
```

The script:
1. Refuses to run on a dirty working tree.
2. Fetches `origin/main`, creates a randomly-named branch (`update-skills-<timestamp>-<rand>`).
3. Wipes `./skills/` and copies every skill folder from `$SOURCE` that has a `SKILL.md`.
4. Regenerates `.claude-plugin/marketplace.json` with the new skill list.
5. Regenerates this README with the current index.
6. Commits, pushes to origin, and prints the `gh pr create` command.

## Install

```bash
/plugin marketplace add arthrod/conejo-skills
/plugin install arthrod-skills
```

## Notable design decisions

- **Stitch-first mandate** — every design-family skill has a `STITCH-DESIGN.md`
  reference and a workflow that drafts to Google Stitch before generating
  final code. See `impeccable/SKILL.md` for the full pattern.
- **Conejo as PR-comment handler** — `conejo` has two modes: **skeptical**
  (hunt PRs, file CR-plan issues, TDD-implement) and **calm-implement**
  (gate each comment, group into tasks, test/install/ship, push back on
  bad comments). The zanahoria family stress-tests plans before
  implementation.
- **`ui-ux-pro-max` is last-resort** — invoked only when stuck on UX or the
  user expresses frustration. For normal design work, `impeccable` is the
  default entry point.
- **Cross-references** — most skills in a family carry a `## See also`
  block linking siblings (e.g. `pi-using` ↔ `pi-extending`, all zanahoria
  variants, all better-auth-*, etc.).
